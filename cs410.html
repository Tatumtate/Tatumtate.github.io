Narrative

Artifact Description

The artifact selected for this enhancement is a C++ program that I previously analyzed and reverse engineered as part of earlier coursework. The original program contained several structural and security weaknesses, including poor organization, lack of modular design, and limited input validation.

Justification for Inclusion

I selected this artifact because it demonstrates my ability to analyze existing code and apply meaningful improvements. Rather than creating a new program, this project highlights my ability to evaluate and enhance software.

  • Refactored the program to reduce reliance on global variables
  • Improved modular design through structured functions
  • Added input validation to prevent unintended behavior
  • Enhanced readability and maintainability

These changes transformed the program into a more structured and reliable application, demonstrating my growth in software design and development practices.

Course Outcomes Alignment

This enhancement demonstrates my ability to design and evaluate computing solutions by identifying weaknesses and implementing improvements. It also reflects my understanding of software engineering principles such as modular design, maintainability, and secure coding.

Reflection on the Enhancement Process

Enhancing this artifact reinforced the importance of writing code that is not only functional but also maintainable and secure. One challenge I faced was restructuring the program without breaking its original functionality, which required careful testing and attention to detail.

Overall, this process strengthened my understanding of clean code practices and the importance of continuous improvement in software development.
